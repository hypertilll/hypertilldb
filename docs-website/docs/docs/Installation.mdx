# Installation

Hypertill DB is published on npm as `@hypertill/db`.

:::tip Fastest way to validate the package
If you want to see the package working before wiring it into your own app, open the TypeScript Expo demo:

- [expo-hypertillDB-example](https://github.com/hypertilll/expo-hypertillDB-example)
:::

## Install the package

For most apps, start with npm:

```bash
npm install @hypertill/db
```

If you already know you are building with Expo, install the dev client too:

```bash
npm install @hypertill/db expo-dev-client
```

## Recommended path: Expo SDK 54+

Hypertill DB `0.0.3` ships an Expo config plugin in the package. That is the easiest current way to get Android JSI wiring and native SQLite linked without manually editing Gradle files.

### 1. Register the package plugin

Add `@hypertill/db` to your Expo plugins list:

```json
{
  "expo": {
    "plugins": ["@hypertill/db", "expo-dev-client"]
  }
}
```

### 2. Babel is auto-configured by the package plugin

When `@hypertill/db` runs as an Expo plugin, it automatically patches `babel.config.js` with the required decorator/class-field plugin chain (in the right order).

You can still manage Babel manually if you want full control:

```js
module.exports = function (api) {
  api.cache(true)
  const resolveFromHypertill = (plugin) =>
    require.resolve(plugin, { paths: [require.resolve('@hypertill/db/package.json')] })

  return {
    presets: ['babel-preset-expo'],
    plugins: [
      [resolveFromHypertill('@babel/plugin-proposal-decorators'), { legacy: true }],
      [resolveFromHypertill('@babel/plugin-proposal-class-properties'), { loose: true }],
      [resolveFromHypertill('@babel/plugin-proposal-private-methods'), { loose: true }],
      [resolveFromHypertill('@babel/plugin-proposal-private-property-in-object'), { loose: true }],
    ],
  }
}
```

To opt out of automatic Babel patching:

```json
{
  "expo": {
    "plugins": [["@hypertill/db", { "disableAutoBabelConfig": true }], "expo-dev-client"]
  }
}
```

### 3. Build a native client

```bash
npx expo prebuild
npx expo run:android
npx expo run:ios
```

### 4. Important note about Expo Go

Hypertill DB uses native SQLite and JSI modules. That means Expo Go is not enough for this package. Use a dev build created through `expo run:*` or your own EAS build.

## Bare React Native

If you are not using Expo, the package still works as a regular React Native native module.

### 1. Install the package

```bash
npm install @hypertill/db
```

### 2. Enable decorators

```js
module.exports = function (api) {
  api.cache(true)
  const resolveFromHypertill = (plugin) =>
    require.resolve(plugin, { paths: [require.resolve('@hypertill/db/package.json')] })

  return {
    presets: ['module:metro-react-native-babel-preset'],
    plugins: [
      [resolveFromHypertill('@babel/plugin-proposal-decorators'), { legacy: true }],
      [resolveFromHypertill('@babel/plugin-proposal-class-properties'), { loose: true }],
      [resolveFromHypertill('@babel/plugin-proposal-private-methods'), { loose: true }],
      [resolveFromHypertill('@babel/plugin-proposal-private-property-in-object'), { loose: true }],
    ],
  }
}
```

### 3. Install iOS pods

```bash
cd ios
pod install
```

### 4. Run your app

```bash
npx react-native run-android
npx react-native run-ios
```

Autolinking should handle the base native package automatically on modern React Native.

<details>
  <summary>Manual Android JSI setup for bare apps</summary>

If you are not using the Expo plugin and want synchronous JSI mode on Android, add the extra native project manually.

In `android/settings.gradle`:

```gradle
include ':watermelondb-jsi'
project(':watermelondb-jsi').projectDir =
    new File(rootProject.projectDir, '../node_modules/@hypertill/db/native/android-jsi')
```

In `android/app/build.gradle`:

```gradle
android {
  packagingOptions {
    pickFirst '**/libc++_shared.so'
  }
}

dependencies {
  implementation project(':watermelondb-jsi')
}
```

In `android/app/proguard-rules.pro`:

```text
-keep class com.hypertill.db.** { *; }
```

Then register the package in `MainApplication.kt` or `MainApplication.java`:

```kotlin
import com.hypertill.db.jsi.HypertillDBJSIPackage

override fun getPackages(): List<ReactPackage> {
  return PackageList(this).packages.apply {
    add(HypertillDBJSIPackage())
  }
}
```

The Gradle subproject name remains `:watermelondb-jsi` in `0.0.3` for compatibility, even though the Java package names are rebranded to `com.hypertill.db.*`.

</details>

## Web and Node.js

Hypertill DB is not limited to native mobile apps:

- Use `SQLiteAdapter` in Node.js
- Use `LokiJSAdapter` in the browser

The database bootstrap pattern is the same either way. The only thing that changes is the adapter implementation. Continue with [Setup](./Setup) for the project structure.

## Verify the install

Before building your real feature set, confirm these basics:

- the app boots on Android and iOS
- your `onSetUpError` callback does not fire
- you can create one record and read it back
- your app is wrapped with `DatabaseProvider`

Then move on to [Setup](./Setup) and [Connecting Components](./Components).
